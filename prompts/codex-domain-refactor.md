# Prompt para mejorar un dominio

Tarea: mejorar documentación de un dominio específico de `domains/`.

Instrucciones:

- Lee el archivo del dominio.
- Lee contratos y niveles de riesgo.
- Propón campos mínimos, flujos, confirmaciones y posibles herramientas.
- No conviertas el dominio en una app tradicional.
- Mantén separación Markdown vs BD/API vs LLM.

Salida:

- Diagnóstico.
- Propuesta de estructura.
- Cambios sugeridos.
- Preguntas para Néstor.
